# Отчет по практической работе

## Цель работы

Ознакомиться с построением маршрутов в Django и реализовать маршрутизацию web-приложения магазина футбольной атрибутики.

## Ход выполнения

1. В приложении `firstproject` были созданы модели для 8 таблиц:
   `Category`, `Club`, `Brand`, `Supplier`, `Product`, `Customer`, `Order`, `OrderItem`.

2. Для таблиц `Product`, `Category`, `Club`, `Brand`, `Supplier`, `Order` реализованы страницы списка и подробного просмотра с использованием generic views:
   `ListView`, `DetailView`, `TemplateView`.

3. В файле маршрутизации приложения `firstproject/urls.py` добавлены именованные маршруты:
   главная страница, информация о компании, товары, категории, клубы, бренды, поставщики, заказы и корзина.

4. Для всех страниц созданы HTML-шаблоны. Общий дизайн вынесен в базовый шаблон `base.html`, а стили находятся в `static/firstproject/css/styles.css`.

5. Для товаров, клубов и брендов добавлены изображения в папку `static/firstproject/img`. На страницах они выводятся через Django-теги `{% static %}`.

   Источники фотографий товаров:

   - Домашняя форма Спартак: `https://soccer-store.ru/images/products/7405/large/1635937118.jpg`.
   - Шарф Барселоны: `https://shop.atributika.ru/upload/iblock/e42/hf8quzx1dp3kpnrwlukvpz1iawici0fp.jpg`.
   - Футбольный мяч Adidas Trionda Pro ЧМ 2026: `https://www.football-mania.ru/f/product/jd8021_02.jpg`.
   - Футбольные бутсы Adidas Predator Freak.1 FG: `https://footballroom.ru/upload/iblock/16a/16a094a45de91e2355a3032c13adaa63.jpg`.

6. Создана и применена миграция `0001_initial.py`, после чего база данных была заполнена демонстрационными данными командой:

```bash
../.venv/bin/python manage.py seed_football_shop
```

7. Проверка маршрутов выполнена через Django test client. Все основные страницы возвращают статус `200 OK`.

## Вывод

В ходе работы было создано Django-приложение для магазина футбольной атрибутики. Были описаны модели базы данных, настроены маршруты внутри приложения, реализованы страницы списков и детальной информации через generic views. Данные выводятся в HTML-шаблонах с помощью Django-тегов, а единый внешний вид обеспечивается базовым шаблоном и общим CSS-файлом.
